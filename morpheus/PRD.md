# Morpheus — 产品需求文档 (PRD)

## 1. 产品概述

**Morpheus** 是一个轻量化的梦境记录 Web 应用。用户醒来后通过语音快速记录梦境内容，AI 自动将零散的口述整理为结构化的故事摘要，并保存到 Google Sheets 中。

### 目标用户
- 经常做梦且希望记录梦境内容的人
- 对梦境中的创意素材（写作、艺术、游戏设计）感兴趣的人
- 希望追踪自己梦境模式的用户

### 核心价值
- **轻**：刚醒时脑子糊，语音一键记录，不需要打字
- **智**：AI 自动提炼故事摘要和标签，无需手动整理
- **连**：通过 Google Sheets 存储，数据属于用户自己

## 3. 数据结构

### 3.1 记录格式

每条梦境记录包含以下字段：

```json
{
  "date": "2026-05-09",
  "tags": {
    "mood": "恐怖/惊悚",
    "themes": ["恐怖", "密室逃脱", "轮回"],
    "elements": ["蟑螂", "蟑螂鼠", "狭窄通道"]
  },
  "summary": "一个恐怖版的密室逃脱游戏。蟑螂群聚成老鼠的形状，诡异而令人不安。在第四关时独自绕圈出不去，最终决定倒着往回走，爬过狭窄通道回到起点。",
  "raw": "昨晚做梦又梦见素材了...像密室逃脱真恐怖版本。那个密室逃脱那个环境真的非常恶劣，里面有蟑螂，各种乱七八糟的一些脏东西。"
}
```

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| `date` | string (YYYY-MM-DD) | 记录日期 | 系统自动生成（当天） |
| `tags.mood` | string | 情绪/氛围（恐惧、温暖、焦虑、奇幻等） | AI 提取 |
| `tags.themes` | string[] | 主题标签（如"密室逃脱"、"轮回"、"飞行"） | AI 提取 |
| `tags.elements` | string[] | 元素标签（如"蟑螂"、"老鼠"、"狭窄通道"） | AI 提取 |
| `summary` | string | AI 润色后的故事摘要，1-2 句话，有故事感 | AI 生成 |
| `raw` | string | 语音识别的原始文字 | 语音识别直接输出 |

### 3.2 Google Sheets 表结构

| 列名 | 对应字段 | 说明 |
|------|----------|------|
| A | Date | 日期 |
| B | Mood | 情绪 |
| C | Themes | 主题（逗号分隔） |
| D | Elements | 元素（逗号分隔） |
| E | Summary | 故事摘要 |
| F | Raw | 原始记录 |

## 4. 用户流程

### 4.1 首次使用

```
打开网页
  → 看到 Morpheus logo 和录音按钮（可用状态，无需先绑定 Google）
  → 点击「设置」→ 点击「绑定 Google」
  → 跳转 Google 授权页面
  → 用户同意授权
  → 返回应用，自动创建「Morpheus Dreams」Spreadsheet
  → 显示「Google 账号已绑定」
  → 回到首页

注：用户未绑定 Google 时仍可录音和 AI 解析，
    但在点击「保存到 Google Sheets」时弹出提示引导绑定。
```

### 4.2 记录梦境

```
点击录音按钮（麦克风图标变为红色，开始录音）
  → 进入录音界面（波形动画 + 计时器）
  → 用户口述梦境内容
  → 录音结束条件（满足任一即停止）：
      a. 用户点击红色停止按钮
      b. 计时器达到 3 分钟上限，自动停止并提示"已达录音上限"
  → 进入「AI 解析中」页面（转圈动画，约 2-3 秒）
  → 进入「确认记录」页面，显示：
      - 日期（自动填入当天，可修改）
      - 标签（mood/themes/elements，彩色标签，可增删改）
      - 故事摘要（AI 润色版，可编辑文本）
      - 原始记录（语音识别原文，可编辑文本）
  → 用户确认无误 → 点击「保存到 Google Sheets」
  → 如未绑定 Google → 弹出提示「请先绑定 Google 账号」→ 跳转设置页
  → 显示「已保存」成功页面
  → 点击「继续记录」回到首页

  → 用户不想保存 → 点击「放弃」按钮
  → 弹出确认弹窗「确定放弃本次记录？」
  → 确认后返回首页，数据不保存
```

### 4.3 查看历史

```
点击底部「故事集」
  → 优先读取本地缓存（localStorage），秒开列表
  → 后台静默调用 Google Sheets API 检查是否有新数据
  → 如有更新则刷新列表并同步本地缓存
  → 显示梦境记录列表（按日期倒序）
  → 每条记录显示：日期 + 故事摘要 + 标签
  → 首次使用显示空状态（月亮图标 + "还没有梦境记录"）

数据缓存策略：
  → 每次成功保存时，同步写入 localStorage 一份副本
  → 打开故事集时先展示本地数据，再后台同步
  → 如本地缓存为空且已绑定 Google，则从 Sheets 全量拉取一次
```

### 4.4 AI 解析异常处理

```
录音结束 → 语音识别返回文字
  → 如果识别文字为空或极短（< 5 字）：
      → 提示「录音内容太少，请重新记录」→ 返回首页
  → 如果 AI 解析超时（> 10 秒）或返回格式错误：
      → 提示「AI 解析失败」
      → 提供两个选项：
          a.「重试」→ 重新调用 AI
          b.「手动填写」→ 进入确认编辑页，仅填入 raw 字段，
             其余字段由用户手动填写
  → 如果 AI 返回结构完整但内容质量明显不合理：
      → 正常进入确认编辑页，用户自行修改
```

### 4.5 网络异常处理

```
录音完成但无网络：
  → 语音识别：Web Speech API 需要网络，无网时录音按钮不可点击
  → 检测到离线状态时，首页显示提示条「需要网络连接才能使用语音识别」

AI 解析或保存时网络断开：
  → 显示「网络连接失败」提示
  → 提供「重试」按钮
  → 已解析的数据暂存在内存中（不丢失），网络恢复后可重试保存

Google Sheets 写入失败：
  → 提示「保存失败，请检查网络或 Google 账号授权状态」
  → 提供「重试」和「复制到剪贴板」两个选项
  → 用户可手动粘贴到其他地方作为备份
```

## 6. 技术架构

### 6.1 前端

- **技术栈**：纯 HTML + CSS + JavaScript（无框架）
- **部署**：Cloudflare Pages
- **域名**：待确定

### 6.2 语音识别

- **方案**：Web Speech API（浏览器自带 `SpeechRecognition`）
- **语言**：中文（`zh-CN`）
- **兼容**：Chrome / Safari 桌面和移动端
- **交互模式**：点击开始录音 → 点击停止录音（非按住说话）
- **时长限制**：最长 3 分钟，到达上限自动停止并给出提示
- **降级方案**：浏览器不支持 Web Speech API 时，显示文字输入框供用户手动键入梦境内容

### 6.3 AI 解析

- **模型**：待确定（使用统一的 API Key，由开发者管理，通过 Cloudflare Worker 中转）
- **Prompt 设计**：将原始语音文字发送给 AI，返回结构化 JSON
- **响应格式**：严格按上述数据结构返回 JSON
- **超时设置**：请求超时 10 秒
- **容错机制**：
  - 输入文字 < 5 字时跳过 AI 解析，直接提示用户内容太少
  - AI 返回非法 JSON 时，自动重试 1 次
  - 重试仍失败则进入手动填写模式（仅保留 raw 字段）
  - 前端对 AI 返回值做 schema 校验，缺失字段填入默认值（如 mood 默认为"未分类"）

### 6.4 存储

- **方案**：Google Sheets API v4
- **认证**：Google OAuth 2.0（用户绑定自己的 Google 账号）
- **数据归属**：数据存储在用户自己的 Google Sheets 中
- **Sheet 创建逻辑**：
  - 用户首次绑定 Google 账号后，应用自动在用户的 Google Drive 中创建名为 `Morpheus Dreams` 的 Spreadsheet
  - 自动写入表头行（Date / Mood / Themes / Elements / Summary / Raw）
  - Sheet ID 存储在浏览器 localStorage 中，后续直接写入该 Sheet
  - 如果检测到 Sheet 被用户删除（API 返回 404），提示用户并自动重建新 Sheet
- **解绑处理**：
  - 用户在设置页可解绑 Google 账号
  - 解绑时清除本地存储的 OAuth token 和 Sheet ID
  - 已保存到 Google Sheets 的数据不受影响（数据属于用户）
  - 重新绑定后，创建新的 Sheet（不自动关联旧 Sheet）

### 6.5 OAuth 配置

- **项目**：单个 Google Cloud 项目
- **模式**：Testing 模式（添加测试用户白名单，上限 100 人）
- **授权范围**：
  - `https://www.googleapis.com/auth/spreadsheets`（读写 Sheets）
  - `https://www.googleapis.com/auth/drive.file`（创建新文件）
- **生产化路径**：如需超过 100 用户，需提交 Google OAuth 验证审核

## 7. 非功能需求

### 7.1 性能

- 首页加载时间 < 2 秒
- AI 解析响应时间 < 5 秒
- 页面切换无明显卡顿

### 7.2 兼容性

- **浏览器**：Chrome（推荐）、Safari（iOS/macOS）
- **语音功能限制**：Web Speech API 在 Firefox 中不受支持，Firefox 用户将看到提示「请使用 Chrome 或 Safari 以获得最佳体验」，并提供手动文字输入作为降级方案
- **设备**：手机浏览器为主，桌面浏览器为辅
- **屏幕**：支持 320px 及以上宽度

### 7.3 隐私

- 梦境记录存储在用户自己的 Google Sheets 中
- 语音识别通过浏览器 Web Speech API 处理，音频数据不经过本应用服务器（由浏览器厂商处理）
- AI 解析时仅发送文字内容，不发送音频
- 不收集用户个人信息

### 7.4 安全

- **API Key 保护**：AI API Key 不暴露在前端代码中，通过 Cloudflare Worker 中转请求
- **OAuth token 存储**：存储在浏览器 localStorage 中
  - v0.1 接受此方案（单用户 + 个人使用）
  - 后续版本考虑迁移到 httpOnly cookie + Cloudflare Worker session 以防范 XSS
- **HTTPS**：所有请求强制 HTTPS（Cloudflare Pages 默认支持）
- **输入校验**：AI 返回的 JSON 在前端做结构验证，防止非法数据写入 Sheets
